# 오디오 스위처 (Audio Switcher) 기획서

## 1. 개요

스피커/헤드셋/이어폰(재생 장치)과 마이크(녹음 장치)를 GUI 클릭이나 글로벌 단축키로
빠르게 전환할 수 있는 Windows용 트레이 상주 프로그램. 전환 시 Console/Multimedia/
Communications 세 가지 역할을 모두 갱신하여, 디스코드처럼 "통신 기본 장치"만 별도로
보는 앱에서도 확실히 반영되게 한다. 나아가 특정 앱(디스코드 등)을 지정해 항상 정해진
장치를 따라가게 고정하는 기능까지 포함한다.

- SoundSwitch, nircmd 기반 배치 스크립트 등을 참고했으나 그대로 가져다 쓰지 않고
  전체를 처음부터 새로 구현한다 (SoundSwitch는 GPLv2라 코드를 그대로 쓰면 배포 시
  소스 공개 의무가 생기기 때문).
- 최종 산출물은 PyInstaller로 묶은 단일 실행 파일(.exe).

## 2. 핵심 기능

1. **개별 전환**: 재생 장치, 녹음 장치를 각각 목록에서 골라 즉시 전환
2. **프로필**: 재생+녹음 장치 조합을 이름 붙여 저장, 한 번에 전환
   - 이름 자유 지정/수정 가능, 개수 제한 없음
   - 프로필별 아이콘 지정(자동 추천 + 수동 변경)
   - 프로필별 글로벌 단축키 지정 가능 (예: `ctrl+alt+1`)
3. **장치 별칭**: Windows가 붙인 지저분한 장치 이름(예: "Headset (2- USB Audio
   Device)") 대신 앱 안에서만 보이는 별명(예: "게이밍 헤드셋")을 지정. 실제 장치 ID는
   그대로 유지되고 표시 이름만 바뀜.
4. **앱별 장치 고정(디스코드 등)**: 지정한 프로세스(디스코드, 줌 등)가 항상 특정
   재생/녹음 장치를 쓰도록 고정. 전환 시 해당 앱도 같이 따라가게 함.
5. **트레이 상주**: 창을 닫아도 종료되지 않고 트레이에 남음. 트레이 아이콘 클릭 시
   팝업으로 프로필/장치 목록이 뜨고 바로 전환 가능.
6. **다중 장치 확장**: 헤드셋 여러 개, 이어폰, 추가 마이크 등이 연결되면 목록에
   자동으로 반영되어 별도 설정 없이 확장 가능.

## 3. UI 구성

### 3-1. 트레이 팝업 (핵심 UX)
작업표시줄 트레이 아이콘 클릭 시 우측 상단에 뜨는 팝업.

- **프로필 영역**: 저장된 프로필들이 아이콘+이름 버튼으로 나열, 마지막에 "추가"
  버튼. 처음 실행 시엔 프로필이 없으므로 "저장된 프로필 없음 · 눌러서 추가" 안내만
  표시됨.
- **재생 장치 영역**: 연결된 재생 장치 목록, 현재 선택된 장치는 강조 표시(파란
  배경 + 체크)
- **녹음 장치 영역**: 연결된 녹음 장치 목록, 동일한 방식으로 강조 표시
- 상태 배지 등 불필요한 요소는 넣지 않고 최대한 깔끔하게 유지

### 3-2. 메인 창 (트레이 팝업과 별도로, 앱을 직접 열었을 때)
- **개별 전환 탭**: 재생/녹음 장치 콤보박스 + 전환 버튼, 새로고침 버튼
- **프로필 탭**: 프로필 목록(표 형태) + 추가/수정/삭제, 프로필 추가 시 재생+녹음
  장치 선택, 단축키 입력, 아이콘 선택, "이 프로필 전환 시 따라갈 앱" 체크박스
- **장치 별칭 설정**: 장치 목록에서 별명 지정/수정

## 4. 기술 스택

- **언어**: Python 3.10+
- **오디오 제어**: `pycaw` (장치 열거), `comtypes` (저수준 COM 인터페이스 직접 호출)
- **기본 장치 전환**: `IPolicyConfig` (SetDefaultEndpoint, 3-role 순회) — 이미 구현됨
- **앱별 장치 고정**: `Windows.Media.Internal.AudioPolicyConfig` (WinRT 비공식 COM,
  vtable 오프셋 호출) — 신규 구현 필요, 리스크 있음 (아래 6번 참고)
- **GUI**: `tkinter` (기본 내장, 추가 설치 불필요)
- **트레이 아이콘**: `pystray` + `Pillow`
- **글로벌 단축키**: `keyboard`
- **설정 저장**: JSON (`%APPDATA%\AudioSwitcher\`)
- **배포**: `PyInstaller` (`--onefile --windowed`)

## 5. 개발 진행 순서 (마일스톤)

- **Phase 1 — 기본 전환 엔진** (완료)
  - 재생/녹음 장치 열거 (`audio_devices.py`)
  - 3-role 동시 전환 (`policy_config.py`)
  - 프로필 저장/불러오기 (`config_manager.py`)
- **Phase 2 — 기본 GUI** (완료, 리팩토링 예정)
  - 개별 전환 탭, 프로필 탭, 트레이 아이콘, 글로벌 단축키 (`main.py`)
- **Phase 3 — UI 개선**
  - 트레이 팝업을 지금 합의한 목업 형태로 재구현 (프로필 아이콘 그리드 + 재생/녹음
    리스트, 배지 제거)
  - 장치 별칭 기능 추가
- **Phase 4 — 앱별 장치 고정 (디스코드 등)**
  - `AudioPolicyConfig` WinRT 인터페이스를 Python(ctypes/comtypes)으로 새로 구현
  - 프로필에 "따라갈 앱" 목록 연결
  - 실패 시 폴백: 안내 메시지로 Windows 자체 설정(앱 볼륨 및 장치 기본 설정) 수동
    고정 안내
- **Phase 5 — 다중 장치 테스트**
  - 헤드셋 2개 이상, 이어폰, 추가 마이크 등 다양한 조합에서 목록 반영/전환 확인
- **Phase 6 — 패키징**
  - PyInstaller로 단일 exe 빌드, 아이콘 적용, 실행 테스트
- **Phase 7 — 안정화**
  - 실제 사용 중 발견되는 버그 수정, 예외 처리 보강

## 6. 리스크 및 대안

- `AudioPolicyConfig`는 Windows 버전마다 내부 GUID/vtable 오프셋이 달라질 수 있는
  비공식 API라 깨질 가능성이 있음. 이 기능이 불안정할 경우, "자동 추적"은 끄고
  "수동 안내"(Windows 설정에서 디스코드를 특정 장치에 고정해두라는 가이드)로 대체.
- 글로벌 단축키(`keyboard` 라이브러리)가 일부 환경(관리자 권한 앱이 포커스인 경우)
  에서 안 먹을 수 있음 → 관리자 권한 실행 안내 필요.

## 7. 목표

- 하나의 exe 실행만으로 스피커/헤드셋/이어폰/마이크 전환이 클릭 한 번 또는 단축키
  한 번으로 끝날 것
- 전환 시 디스코드 등 지정한 앱도 자동으로 같이 따라갈 것
- 장치가 늘어나도(헤드셋 추가 구매 등) 별도 설정 없이 목록에 자동 반영될 것
- 형석님 개인 소유의 오리지널 프로그램으로, 외부 라이선스 제약 없이 자유롭게
  수정/배포 가능할 것
